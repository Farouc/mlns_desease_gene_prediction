# Experiment Report: Interpretable Disease-Gene Prediction

Date: 2026-04-03

## 1) Scope
This report summarizes all work executed so far to make the repository runnable, obtain baseline and CUDA results, and produce a final full run with hybrid enabled for model comparison.

## 2) What Was Done

### 2.1 Data acquisition and preparation
- Downloaded official Hetionet v1.0 archive from Zenodo (`10.5281/zenodo.268568`).
- Extracted raw files into `data/raw/`:
  - `hetionet-v1.0-edges.sif.gz`
  - `hetionet-v1.0-nodes.tsv`
- Built project input file `data/raw/hetionet_subset_edges.csv` filtered to:
  - `Disease`, `Gene`, `Pathway`, and `Phenotype`
- Important mapping used:
  - Hetionet node kind `Symptom` was mapped to project type `Phenotype`.

### 2.2 Environment and dependency setup
- Used `base` environment for all runs.
- Verified core stack: `torch`, `torch_geometric`, `networkx`, `pandas`, `sklearn`, `yaml`, `matplotlib`, `tqdm`.
- Installed missing PyG runtime deps required by Node2Vec:
  - `pyg-lib`
  - `torch-cluster`

### 2.3 Pipeline/debug fixes applied
- Fixed split generation global-id merge issue:
  - `src/data/split.py`
- Fixed NetworkX 3 serialization compatibility (pickle-based graph save/load):
  - `src/graph/build_graph.py`
  - `src/main.py`
  - `src/training/train_node2vec.py`
- Improved HAN training runtime (one full graph forward/backward per epoch instead of per-mini-batch full-forward recomputation):
  - `src/training/train_han.py`
- Accelerated hybrid metapath counting (sparse matrix propagation path counts + caching):
  - `src/graph/metapaths.py`
  - `src/main.py` (passes `node_counts` into metapath counter)

### 2.4 Executed runs
- `smoke_heuristics`
- `n2v_full_cuda`
- `han_full_cuda`
- `tiny_full_results_v2` (fast compact end-to-end check)
- `final_full_cuda_hybrid_e15_fast` (final comparison run, full data, CUDA, hybrid enabled)

## 3) Main Results

### 3.1 Final full comparison run
Run folder: `experiments/results/final_full_cuda_hybrid_e15_fast`

| Model      | AUC-ROC | AUC-PR | Hits@10 | MRR |
|------------|--------:|-------:|--------:|----:|
| Heuristics | 0.7525  | 0.4251 | 0.8309  | 0.5857 |
| Node2Vec   | 0.8498  | 0.4938 | 0.8603  | 0.5866 |
| HAN        | 0.8248  | 0.4573 | 0.6397  | 0.3550 |
| Hybrid     | 0.8495  | 0.5192 | 0.8088  | 0.6287 |

Observations:
- `Hybrid` achieved the best **AUC-PR** and **MRR**.
- `Node2Vec` had the best **AUC-ROC** and **Hits@10** (slightly higher than Hybrid on Hits@10).
- HAN alone underperformed Node2Vec/Hybrid on ranking metrics in this final setting.
